# Aprendizajes

## Lecciones del Proceso

### Técnicos

#### 1. El Research Justifica Decisiones

> **"Sin investigación, el diseño es solo opinión."**

Antes de este proyecto, tomaba decisiones de diseño basadas en "me gusta". El benchmark y las entrevistas me dieron argumentos sólidos:

* El dark mode no es capricho, es tendencia validada (89% de referencias)
* La estructura one-page no es pereza, es comportamiento de usuario
* Los 30 segundos no son exageración, es presión real de reclutadores

**Impacto:** Cada decisión de diseño pudo ser defendida con datos.

***

#### 2. Mobile-First es No Negociable

| Fase        | Enfoque       | Resultado                   |
| ----------- | ------------- | --------------------------- |
| Inicio      | Desktop-first | Mobile fue un afterthought  |
| Aprendizaje | Mobile-first  | Diseño más robusto en ambos |

El cambio de mentalidad fue crucial:

* **Antes:** "Adaptemos a mobile después"
* **Después:** "Si funciona en mobile, funciona en desktop"

**Impacto:** Mejor UX en todos los dispositivos, menos deuda técnica. Quedamos a la deuda, pero lo tenemos identificado.

***

#### 3. Vanilla tiene su Lugar

```
Framework (React/Vue/Angular)
   ↓
Bueno para: Apps complejas, estado, interactividad

Vanilla (HTML/CSS/JS)
   ↓
Bueno para: Portafolios, landing pages, performance
```

Para un portafolio estático:

* ✅ Sin build step
* ✅ Sin dependencies
* ✅ Carga instantánea
* ✅ Control total

**Impacto:** 95 Performance Lighthouse sin esfuerzo de optimización.

***

### De Diseño

#### 4. Consistencia > Creatividad

| Iteración | Enfoque                                 | Feedback      |
| --------- | --------------------------------------- | ------------- |
| v1        | Muchos colores y animaciones            | "Saturado"    |
| v2        | Paleta restringida, animaciones sutiles | "Profesional" |

**Aprendizaje:** La creatividad debe servir al contenido, no eclipsarlo.

***

#### 5. El Espacio es un Elemento de Diseño

```
Antes:
┌─────────────────────────────────────┐
│Elemento│Elemento│Elemento│Elemento││
│Elemento│Elemento│Elemento│Elemento││
└─────────────────────────────────────┘
"Información densa, abrumador"

Después:
┌─────────────────────────────────────┐
│                                     │
│   Elemento     Elemento             │
│                                     │
│        Elemento     Elemento        │
│                                     │
└─────────────────────────────────────┘
"Respira, jerarquía clara"
```

**Impacto:** +20% en percepción de profesionalismo.

***

#### 6. Accesibilidad no es Opcional

| Aspecto        | Implementación | Beneficio            |
| -------------- | -------------- | -------------------- |
| Contraste      | WCAG AA        | Todos pueden leer    |
| Keyboard nav   | Tab navigation | Sin mouse funciona   |
| Alt text       | Descripciones  | Screen readers       |
| Reduced motion | Media query    | Respeto preferencias |

**Impacto:** Lighthouse Accessibility 100/100. Es lo correcto y es buen negocio.

***

### Personales

#### 7. Mi Voz Importa

> **"El portafolio genérico no diferencia. El auténtico conecta."**

Integrar el tema de videojuegos fue riesgoso:

* ❌ Miedo: "¿Parecerá infantil?"
* ✅ Realidad: "Muestra personalidad sin sacrificar profesionalismo"

**Aprendizaje:** La autenticidad es un diferenciador. No imites, inspírate.

***

#### 8. Iterar es Progreso

```
v1 → Feedback → v2 → Feedback → v3 → Final
 │              │              │
 └──────────────┴──────────────┘
         Mejora continua
```

El testeo con usuarios reales reveló:

* Lo que yo creía importante ≠ Lo que ellos necesitan
* Pequeños cambios = Grandes mejoras en UX
* El "perfecto" es enemigo del "listo para usar"

***

#### 9. Documentar es Parte del Proceso

Este GitBook no es "tarea extra", es:

* **Reflexión:** Pensar en lo aprendido
* **Comunicación:** Mostrar proceso a otros
* **Portafolio:** Evidencia de pensamiento de diseño

**Impacto:** Ahora tengo case study completo, no solo screenshots.

***

## Errores y Correcciones

| Error                       | Consecuencia           | Corrección                   |
| --------------------------- | ---------------------- | ---------------------------- |
| Ignorar mobile inicialmente | Rediseño significativo | Mobile-first desde el inicio |
| Demasiados colores          | Diseño saturado        | Paleta restringida           |
| Sin testing temprano        | Cambios tardíos        | Validar con usuarios en v1   |
| Texto genérico              | Sin diferenciación     | Copy personalizado           |
